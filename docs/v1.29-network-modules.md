# v1.29 Network Modules

The v1.29 network modules add parity coverage for NAT, DHCP, circuits,
location inventory fields, and customer ownership associations.

## API Endpoints

- NAT rules: `GET/POST /api/v1/nat-rules`, `GET/PUT/DELETE /api/v1/nat-rules/{id}`
- DHCP servers: `GET/POST /api/v1/dhcp-servers`, `PUT/DELETE /api/v1/dhcp-servers/{id}`
- DHCP leases: `GET/POST /api/v1/dhcp-leases`, `PUT/DELETE /api/v1/dhcp-leases/{id}`
- Circuit providers: `GET/POST /api/v1/circuit-providers`, `PUT/DELETE /api/v1/circuit-providers/{id}`
- Physical circuits: `GET/POST /api/v1/physical-circuits`, `PUT/DELETE /api/v1/physical-circuits/{id}`
- Logical circuits: `GET/POST /api/v1/logical-circuits`, `PUT/DELETE /api/v1/logical-circuits/{id}`
- Customer associations: `GET/POST /api/v1/customer-associations`, `DELETE /api/v1/customer-associations/{id}`
- Customer-scoped associations: `GET /api/v1/customers/{id}/associations`

All endpoints are additive to the stable v1 API and use existing session/API
token authentication, CSRF protection for browser mutations, feature flags, and
RBAC checks.

## Feature Flags

The modules are enabled by default and can be disabled from admin settings:

- `feature_nat_enabled`
- `feature_dhcp_enabled`
- `feature_circuits_enabled`

Existing customer and location feature flags continue to control customer
associations and location inventory pages.

## Permissions

The new permission families are:

- `ipam:nat:{list,read,write,delete}`
- `ipam:dhcp:{list,read,write,delete}`
- `ipam:circuit:{list,read,write,delete}`

The migration backfills read access to roles with existing IPAM permissions and
write/delete access to roles that can already write subnets.
