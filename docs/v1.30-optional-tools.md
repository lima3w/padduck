# v1.30 Optional Tools Parity

The v1.30 optional tools release adds operator-facing modules that are useful
for network documentation but remain individually feature-gated.

## Firewall Zone Mapping

Firewall zone mapping documents security zones and their relationships to IPAM
objects or explicit CIDR ranges.

### API Endpoints

- Firewall zones: `GET/POST /api/v1/firewall-zones`, `GET/PUT/DELETE /api/v1/firewall-zones/{id}`
- Firewall zone mappings: `GET/POST /api/v1/firewall-zone-mappings`, `PUT/DELETE /api/v1/firewall-zone-mappings/{id}`

`GET /api/v1/firewall-zone-mappings` accepts optional `zone_id` filtering.
Mappings must reference either an IPAM object ID or a CIDR.

### Feature Flag

- `feature_firewall_enabled`

The flag defaults to enabled. Disabling it hides the UI route and returns
`404` from the module API routes.

### Permissions

- `ipam:firewall:{list,read,write,delete}`

The migration backfills read access to roles with existing IPAM permissions and
write/delete access to roles that can already write subnets.
